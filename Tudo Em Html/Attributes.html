accept

O acceptatributo assume como valor uma lista separada por vírgulas de um ou mais tipos de arquivo, ou especificadores de tipo de arquivo exclusivos , descrevendo quais tipos de arquivo permitir.

autocomplete

O autocompleteatributo HTML permite que os desenvolvedores da web especifiquem qual permissão o agente do usuário tem, se houver, para fornecer assistência automatizada no preenchimento dos valores dos campos do formulário, bem como orientar o navegador quanto ao tipo de informação esperada no campo. Está disponível em (input)elementos que utilizam um texto ou valor numérico como entrada, (textarea)elementos, (select)elementos e (form)elementos.

capture

O captureatributo especifica que, opcionalmente, um novo arquivo deve ser capturado e qual dispositivo deve ser usado para capturar essa nova mídia de um tipo definido pelo acceptatributo. Os valores incluem usere environment. O atributo capture é suportado no tipo de entrada do arquivo . O captureatributo toma como valor uma string que especifica qual câmera utilizar para captura de dados de imagem ou vídeo, caso o atributo accept indique que a entrada deve ser de um desses tipos.

crossorigin

O crossoriginatributo, válido nos elementos (audio), (img), (link), (script) e (video), fornece suporte para CORS , definindo como o elemento lida com solicitações de origem cruzada, permitindo assim a configuração das solicitações CORS para os dados buscados do elemento. Dependendo do elemento, o atributo pode ser um atributo de configurações do CORS.

dirname

O dirnameatributo pode ser usado no (textarea)elemento e em vários (input)tipos e descreve a direcionalidade do conteúdo do texto do elemento durante o envio do formulário. O navegador usa o valor desse atributo para determinar se o texto digitado pelo usuário é orientado da esquerda para a direita ou da direita para a esquerda. Quando usado, o valor da direcionalidade do texto do elemento é incluído nos dados de envio do formulário junto com o dirnamevalor do atributo como o nome do campo.

disabled

O atributo booleano disabled, quando presente, torna o elemento não mutável, focável ou mesmo enviado com o formulário. O usuário não pode editar nem focar no controle, nem em seus descendentes de controle de formulário.

elementtiming

O elementtimingatributo é usado para indicar que um elemento está sinalizado para rastreamento por PerformanceObserverobjetos usando o "element"tipo. Para mais detalhes, consulte a PerformanceElementTiminginterface. Este atributo pode ser aplicado a (img) elementos (image) dentro de um svg> imagens de pôster de (video)elementos, elementos que possuem um background-imagee elementos contendo nós de texto, como um (p) . No DOM, esse atributo é refletido como Element.elementTiming.

for

O foratributo é um atributo permitido para (label) e (output) . Quando usado em um (label)elemento, indica o elemento do formulário que este rótulo descreve. Quando usado em um (output)elemento, permite um relacionamento explícito entre os elementos que representam valores usados ​​na saída.

max

O maxatributo define o valor máximo aceitável e válido para a entrada que contém o atributo. Se o valor valuedo elemento for maior que isso, o elemento falhará na validação . Este valor deve ser maior ou igual ao valor do minatributo. Se o maxatributo estiver presente, mas não for especificado ou for inválido, nenhum maxvalor será aplicado. Se o maxatributo for válido e um valor não vazio for maior que o máximo permitido pelo maxatributo, a validação da restrição impedirá o envio do formulário.

maxlength

O maxlengthatributo define o comprimento máximo da string que o usuário pode inserir em um (input)ou (textarea). O atributo deve ter um valor inteiro igual a 0 ou superior. O comprimento é medido em unidades de código UTF-16, que ( para a maioria dos scripts ) é equivalente ao número de caracteres. Se não maxlengthfor especificado ou um valor inválido for especificado, a entrada não terá comprimento máximo.

min

O minatributo define o valor mínimo aceitável e válido para a entrada que contém o atributo. Se o valor valuedo elemento for menor que isso, o elemento falhará na validação . Este valor deve ser menor ou igual ao valor do maxatributo.

minlength

O minlengthatributo define o comprimento mínimo da string que o usuário pode inserir em um (input)ou (textarea). O atributo deve ter um valor inteiro igual a 0 ou superior.

multiple

O atributo booleano multiple, se definido, significa que o controle do formulário aceita um ou mais valores. Válido para os tipos de entrada email e arquivo e (select), a forma como o usuário opta por múltiplos valores depende do controle do formulário.

pattern

O patternatributo especifica uma expressão regular com a qual o valor do controle de formulário deve corresponder. Se um não- nullvalor não estiver em conformidade com as restrições definidas pelo patternvalor, a propriedade ValidityStatesomente leitura do objeto patternMismatchserá verdadeira.

placeholder

O placeholderatributo define o texto exibido em um controle de formulário quando o controle não tem valor. O texto do espaço reservado deve fornecer uma breve dica ao usuário sobre o tipo de dados esperado que deve ser inserido no controle. O texto de espaço reservado eficaz inclui uma palavra ou frase curta que indica o tipo de dados esperado, e não uma explicação ou prompt. O espaço reservado não deve ser usado em vez de um arquivo (label). Como o espaço reservado não é visível se o valor do controle do formulário não for nulo, usar placeholderem vez de um (label)prompt prejudica a usabilidade e a acessibilidade.

readonly

O atributo booleano readonly, quando presente, torna o elemento não mutável, ou seja, o usuário não pode editar o controle.

rel

O relatributo define o relacionamento entre um recurso vinculado e o documento atual. Válido em (link), , e (form), os valores suportados dependem do elemento no qual o atributo é encontrado.

required

O atributo booleano required , se presente, indica que o usuário deve especificar um valor para a entrada antes que o formulário proprietário possa ser enviado.

size

O sizeatributo define a largura (input)e a altura do (select)elemento. Para um inputelemento, define o número de caracteres que o agente do usuário permite que o usuário veja ao editar o valor. Para um selectelemento, define a quantidade de opções que devem ser mostradas ao usuário. Deve ser um número inteiro não negativo válido maior que zero.

step

O stepatributo é um número que especifica a granularidade que o valor ou a palavra-chave deve aderir any. É válido para os tipos de entrada numérica, incluindo os tipos date , month , week , time , datetime-local , number e range .
