< input type="button" >

elementos do tipo buttonsão renderizados como botões simples, que podem ser programados para controlar funcionalidades personalizadas em qualquer lugar de uma página da Web, conforme necessário, quando atribuída uma função de manipulador de eventos (normalmente para o clickevento).

< input type="checkbox" >

elementos do tipo checkboxsão renderizados por padrão como caixas que são marcadas (marcadas) quando ativadas, como você pode ver em um formulário oficial do governo. A aparência exata depende da configuração do sistema operacional em que o navegador está sendo executado. Geralmente é um quadrado, mas pode ter cantos arredondados. Uma caixa de seleção permite selecionar valores únicos para envio em um formulário (ou não).

< input type="color" >

elementos do tipo colorfornecem um elemento de interface do usuário que permite ao usuário especificar uma cor, usando uma interface visual de seleção de cores ou inserindo a cor em um campo de texto em #rrggbbformato hexadecimal.

< input type="date" >

elementos de type="date"criação de campos de entrada que permitem ao usuário inserir uma data. A aparência da interface de entrada do seletor de data varia de acordo com o navegador e o sistema operacional. O valor é normalizado para o formato yyyy-mm-dd. O valor resultante inclui o ano, mês e dia, mas não a hora. Os tipos de entrada local de hora e data e hora suportam entrada de hora e data+hora.

< input type="datetime-local" >

elementos do tipo datetime-localcriam controles de entrada que permitem ao usuário inserir facilmente uma data e uma hora, incluindo o ano, mês e dia, bem como a hora em horas e minutos.

< input type="email" >

elementos do tipo emailsão usados ​​para permitir que o usuário insira e edite um endereço de e-mail ou, se o multipleatributo for especificado, uma lista de endereços de e-mail.

< input type="file" >

elementos type="file"permitem que o usuário escolha um ou mais arquivos do armazenamento do dispositivo. Uma vez escolhidos, os arquivos podem ser enviados para um servidor usando o envio de formulário , ou manipulados usando código JavaScript e a API de Arquivos .

< input type="hidden" >

elementos de tipo hiddenpermitem que os desenvolvedores da web incluam dados que não podem ser vistos ou modificados pelos usuários quando um formulário é enviado. Por exemplo, o ID do conteúdo que está sendo solicitado ou editado no momento ou um token de segurança exclusivo. As entradas ocultas são completamente invisíveis na página renderizada e não há como torná-las visíveis no conteúdo da página.

< input type="image" >

elementos do tipo imagesão usados ​​para criar botões de envio gráficos, ou seja, botões de envio que assumem a forma de uma imagem em vez de texto.

< input type="month" >

elementos do tipo monthcriam campos de entrada que permitem ao usuário inserir um mês e um ano, permitindo que um mês e um ano sejam facilmente inseridos. O valor é uma string cujo valor está no formato " YYYY-MM", onde YYYYé o ano de quatro dígitos e MMé o número do mês.

< input type="number" >

elementos do tipo numbersão usados ​​para permitir que o usuário insira um número. Eles incluem validação integrada para rejeitar entradas não numéricas. O navegador pode optar por fornecer setas de passo para permitir que o usuário aumente e diminua o valor usando o mouse ou tocando com a ponta do dedo.

< input type="password" >

elementos do tipo passwordfornecem uma maneira para o usuário inserir uma senha com segurança. O elemento é apresentado como um controle de editor de texto simples de uma linha no qual o texto é obscurecido para que não possa ser lido, geralmente substituindo cada caractere por um símbolo como o asterisco ("*") ou um ponto ("•" ). Este caractere irá variar dependendo do agente do usuário e do sistema operacional.

< input type="radio" >

elementos do tipo radiosão geralmente usados ​​em grupos de rádio — coleções de botões de rádio que descrevem um conjunto de opções relacionadas. Apenas um botão de opção em um determinado grupo pode ser selecionado ao mesmo tempo. Os botões de opção normalmente são renderizados como pequenos círculos, que são preenchidos ou destacados quando selecionados.

< input type="range" >

elementos do tipo rangepermitem que o usuário especifique um valor numérico que não deve ser menor que um determinado valor e não maior que outro valor determinado. O valor preciso, entretanto, não é considerado importante. Isso normalmente é representado por meio de um controle deslizante ou de discagem, em vez de uma caixa de entrada de texto, como o tipo de entrada numérica . Como esse tipo de widget é impreciso, ele só deverá ser usado se o valor exato do controle não for importante.

< input type="reset" >

elementos do tipo resetsão renderizados como botões, com um clickmanipulador de eventos padrão que redefine todas as entradas no formulário para seus valores iniciais.

< input type="search" >

elementos do tipo searchsão campos de texto projetados para o usuário inserir consultas de pesquisa. Eles são funcionalmente idênticos às textentradas, mas podem ter estilos diferentes pelo agente do usuário .

< input type="submit" >

elementos do tipo submitsão renderizados como botões. Quando o clickevento ocorre (normalmente porque o usuário clicou no botão), o agente do usuário tenta enviar o formulário ao servidor.

< input type="tel" >

elementos do tipo telsão usados ​​para permitir que o usuário insira e edite um número de telefone. Ao contrário de e , o valor de entrada não é validado automaticamente para um formato específico antes que o formulário possa ser enviado, porque os formatos dos números de telefone variam muito em todo o mundo.

< input type="text" >

elementos do tipo textcriam campos de texto básicos de linha única.

< input type="time" >

elementos do tipo timecriam campos de entrada projetados para permitir que o usuário insira facilmente um horário (horas e minutos e, opcionalmente, segundos). Embora a aparência da interface do usuário do controle seja baseada no navegador e no sistema operacional, os recursos são os mesmos. O valor é sempre um horário de 24 horas hh:mmou hh:mm:ssformatado, com zeros à esquerda, independentemente do formato de entrada da UI.

< input type="url" >

elementos do tipo urlsão usados ​​para permitir que o usuário insira e edite um URL.

< input type="week" >

elementos do tipo weekcriam campos de entrada permitindo fácil entrada de um ano mais o número da semana ISO 8601 durante esse ano (ou seja, semana 1 a 52 ou 53 ).
